# Git 常用指令速查：从配置到后悔药

> 面向设计师：这些命令不用背。理解每条「在哪个区域、干什么」，配合上方模块里的**可点击速查**一起看，需要时查得到就行。

如果还不清楚「工作区 / 暂存区 / 仓库」是什么，建议先看《看懂 Git：版本控制入门》。这篇是它的**动手版**——把日常真正会敲的命令按场景列清楚。

## 一次性配置

第一次用 Git，先告诉它「你是谁」，之后每次提交都会署名：

```bash
git config --global user.name "你的名字"
git config --global user.email "you@example.com"
```

`--global` 表示对这台电脑上所有仓库生效，只需配一次。

## 获取一个仓库

两种起点，二选一：

- **全新项目**：`git init` —— 在当前文件夹里建一个空仓库
- **已有项目**：`git clone <仓库地址>` —— 把云端仓库整个下载到本地

```bash
git clone https://github.com/用户名/项目名.git
```

## 日常提交循环

九成时间你只在重复这几条。顺序：看状态 → 加暂存 → 提交。

| 命令 | 作用 |
| --- | --- |
| `git status` | 看现在有哪些改动、哪些已暂存 |
| `git diff` | 看具体改了哪几行（未暂存的） |
| `git add <文件>` | 把某个文件的改动放进暂存区（`git add .` 加全部） |
| `git commit -m "说明"` | 把暂存区的改动存成一个版本，附一句说明 |
| `git log` | 查看提交历史（`git log --oneline` 更精简） |

> commit 的说明（message）要写清「做了什么」，未来的你和同事都会感谢现在的你。

## 与远程同步

本地提交完，还要和云端（远程仓库）互通有无：

| 命令 | 作用 |
| --- | --- |
| `git pull` | 把云端别人的最新改动拉到本地（开工前先拉） |
| `git push` | 把本地的提交上传到云端，别人才看得到 |
| `git fetch` | 只下载云端更新、先不合并（想先看看再说时用） |
| `git remote -v` | 查看当前连接的远程仓库地址 |

## 分支操作

分支 = 一条独立的时间线，做实验不影响主线。

| 命令 | 作用 |
| --- | --- |
| `git branch` | 列出所有分支（带 * 的是当前所在） |
| `git switch <分支名>` | 切换到某条分支（旧写法 `git checkout`） |
| `git switch -c <新分支名>` | 新建并切换到一条新分支 |
| `git merge <分支名>` | 把某条分支的改动合并进当前分支 |

> 改动之前先 `git branch` 确认自己在哪条分支，别在 `main` 上直接动手。

## 后悔药：撤销改动

改错、删错、提交错——几乎都有救。按「错到哪一步」对症下药：

| 场景 | 命令 |
| --- | --- |
| 工作区改乱了，想丢弃还没暂存的改动 | `git restore <文件>` |
| 已经 `git add` 了，想把它撤出暂存区 | `git restore --staged <文件>` |
| 想撤销某次已提交的改动（保留历史） | `git revert <提交号>` |
| 手头活没做完，想先临时收起来 | `git stash` / 恢复用 `git stash pop` |

> `reset` 能改写历史、很猛，新手优先用更安全的 `restore` 和 `revert`；真要用 `reset` 前先问清楚。
