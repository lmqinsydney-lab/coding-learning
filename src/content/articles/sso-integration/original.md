# 给自己的工具接入公司 SSO：从申请到本地跑通

> 面向团队同学：想让你自己做的工具也能用**公司账号登录**、识别用户身份与权限？这篇把「秒创」接入 SSO 的完整过程，整理成一份可复用的步骤——照着做，就能给你的工具接上公司 SSO。

## 为什么要接 SSO

工具想给团队用，第一步是知道「**是谁登录的**」，才能决定这个用户进入什么界面、能操作什么功能。

解决登录有两条路：

| | A 白名单 JSON | B 接入公司 SSO |
| --- | --- | --- |
| 做法 | 项目里放一个 `whitelist.json`，写死谁能用 | 用户身份直接从公司系统调用 |
| 成本 | 5 分钟 | 要读鉴权文档、配回调地址、处理 cookie/session |
| 适合 | 临时 / 个人小工具 | 要长期给团队用的工具 |

> 判断标准不是「短期成本」，而是「**项目能不能脱离我独立运转**」。要长期给团队用，选 B。

## 看懂登录这条链路

整条调用链一句话：**用户点登录 → 跳到 SSO → 登录成功回调你的服务 → 你拿到用户身份**。

一个完整的登录地址长这样，拆开看就不神秘了：

```text
https://me.xiaojukeji.com/project/stargate-auth/html/login.html?redirect_uri=...&app_id=21xxxxx
└─协议─┘ └────────域名 Host────────┘ └──────────路径 Path──────────┘  └ 登录后跳回哪（你的回调）┘  └ 你的应用 ID ┘
```

- **redirect_uri**：登录成功后跳回哪里（你的回调地址）
- **app_id**：标识「是哪个系统在请求登录」——你这个工具的工牌号

接入前建议先补几个概念（不懂就把**脱敏后**的文档丢给 AI，让它用大白话讲一遍）：

1. 接口是什么、由什么构成、怎么调用
2. cookie / session / localStorage 的区别（登录态到底存在哪）
3. 公司 SSO 的那几个接口分别干什么

> 公司内部文档可参考：《接入 SSO 指引》《UPM 使用指引》《SSO 接口文档》。**脱敏后再喂 AI，别贴真实地址和密钥。**

## 申请 app_id 和 app_key

`app_id` / `app_key` 是你的工具在公司系统里的「**工牌**」——有了它，公司才认识你的系统，你才能拿到员工身份信息。它由 **UPM（Unified Permission Management，用户权限管理系统）**发放。

到 **BPM** 提交申请表单（流程 `sso_upm_app_add_v3`），关键字段这样填：

| 字段 | 怎么填 |
| --- | --- |
| 子系统名称 | 你的工具名 |
| 回调地址 | 后端给的 IP + `/auth/callback`；本地是 `http://localhost:8000/auth/callback` |
| 主页地址 | 你的工具首页；本地 `http://localhost:8000` |
| 管理员账号前缀 | 邮箱前缀，英文逗号隔开（线上不能填外包 / 实习生） |
| 系统所属环境 | 测试 / 线上 |
| 机房国家、数据类型等 | 按实际情况下拉选择 |

> 建议**申请两套**：一套本地测试、一套线上正式。测试环境可访问，线上更严格。

申请通过后，邮箱会收到 `app_id` / `app_key`。之后可登录 `upm.xiaojukeji.com`（生产）/ `upm-test.xiaojukeji.com`（测试），在「系统管理员入口 → 系统管理 → 管理员管理」查看修改。

> ⚠️ **你的 `app_key` 绝不要发给任何人或 AI**——一旦泄露，你的平台可能被任何人入侵。

## 配置 .env

AI 生成的项目里会带一个 `.env` 文件（隐藏文件，macOS 按 `⌘ + ⇧ + .` 显示）。它是告诉程序「什么时候调用哪个地址」的导航，也存放关键信息。核对这 5 个值：

| 变量 | 填什么 |
| --- | --- |
| `SSO_APP_ID` | 从 UPM 拿到的 app_id |
| `SSO_APP_KEY` | 从 UPM 拿到的 app_key |
| `SSO_BASE_URL` | 公司 SSO 地址（一般不用改） |
| `APP_BASE_URL` | 你的服务地址（本地默认 `http://localhost:8000`） |
| `SESSION_SECRET` | 会话密钥，终端生成 ↓ |

```bash
openssl rand -base64 32
# 把输出的一串字符填进 SESSION_SECRET
```

## 本地跑通登录

把所有代码放在一个**全英文路径**下（别出现中文）。然后在 IDE 终端启动服务——有 AI 可以「逃课」：直接在终端里 `claude`，对它说「帮我在本地运行服务」即可，登录是很成熟的基础能力，放心交给它。

跑通的标志：

- 终端显示服务已启动、监听 `8000` 端口
- 浏览器打开 `http://localhost:8000`，会**自动跳转到公司 SSO 登录页**
- 登录后跳回你的工具，并能拿到当前用户身份

## 避坑清单

- `app_key` 保密，绝不外发给任何人 / AI
- 连公司内网登录时**别开 VPN**，只连公司内网才行
- 代码存放路径**全英文**，不要出现中文
- 调试三板斧：用无痕模式清缓存、按 `F12` 看「网络 / 控制台」、把**截图 + 完整报错**一起发给 AI
