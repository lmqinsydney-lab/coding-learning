# Coding Learning Platform · PRD / 设计文档

- 文档日期：2026-06-15
- 状态：已与用户对齐核心决策，待评审
- 负责人：用户（设计师，具备设计+coding 背景）
- 一句话定位：把代码 / AI 文章拆解、再生成为「可点击信息图」的学习 Web 平台，专为设计师降低学习门槛。

---

## 1. 背景与目标

团队鼓励设计师学习代码与 AI 知识。用户具备设计+coding 背景，希望建立一个学习平台，把团队成员的文章内容做**结构化拆解与再生成**，让设计师更容易理解代码/AI 知识。

文章内容分两类（一期）：

- **代码类**：用户自有或网上整理的代码文章，需站在设计师视角组织阅读与学习。
- **AI 学习类**：团队成员发布的 AI 学习/探索文章，或制作的好玩插件。

后续可能扩展业务类、设计类等分类（**非一期范围**）。

---

## 2. 核心决策（已与用户对齐）

| 维度 | 决策 |
|---|---|
| 模式 | 混合制：一期由 Claude 手动拆解+生成交互页，成员只「看/学」；二期做自助上传+AI 自动生成 |
| 一期形态 | 纯前端可跑的 Web 原型（`npm run dev` 即开），不接后端、不登录、不接 API |
| 最终形态 | 一个会持续长大的 Web 端产品 |
| 生成引擎 | LLM 生成可交互 HTML/SVG（非文生图位图）；效果不足再考虑文生图兜底 |
| 一期生成方式 | 由 Claude 手动产出文章数据与交互页 |
| 内容分类 | 仅「代码类」「AI 学习类」 |
| 架构 | 方案 C：共享外壳 + 可插拔模块 |
| 技术栈 | React + Vite + TypeScript |
| UI 风格 | 暗色高级感（参考 Linear 风，紫色强调） |

## 3. 一期范围（Scope）

**做：**

- 目录页（暗色卡片网格 + 分类筛选）
- 文章总览页（内容模块大纲 + 原文铺底）
- 模块详情右侧抽屉（连续滚动 + 解读视角锚点 + 专属交互）
- 可复用解读视角组件库 + 核心交互范式库
- 至少 2 篇样板文章（1 篇代码类 Git 文章 + 1 篇 AI 类）

**不做（YAGNI / 非目标）：**

- 后端、数据库、登录与权限
- API 调用、文生图
- 自助上传（Word/PDF/Cooper 链接）的解析与自动生成
- 部署上线
- 评论/收藏等社交功能
- 业务类/设计类分类

## 4. 信息架构与页面

内容采用**两层模型**：第一层是文章自身的内容大纲，第二层是每个内容模块再生成的结构化学习层。

1. **目录页** `/`
   - 暗色卡片网格；筛选 `全部 / 代码类 / AI 学习类`
   - 每张卡片：标题、分类、内容模块数、交互亮点标签、摘要

2. **文章总览页（基页）** `/article/:id`
   - **第一层 = 内容模块大纲**：提取文章**最高层级标题**作为几大核心内容模块，平铺成信息图，一眼看懂文章结构。
   - 大纲**下方直接铺原文**，供通读参考；内容模块充当原文各章节的锚点/入口。
   - 点某个内容模块 → 从右侧滑出抽屉（见下）。

3. **模块详情（右侧抽屉，非新页面）**
   - 总览页**始终作底**；点内容模块从右侧滑出抽屉，展示该模块**再生成的结构化学习层**（讲解/图解/交互），关掉即回总览，上下文不丢。
   - 抽屉是**一篇连续滚动的内容**；顶部是「解读视角」锚点导航（**导读 / 核心概念图解 / 指令交互速查 / 避坑**，按需取用），点击跳转、滚动时自动高亮当前锚点（scrollspy），不是切 tab 面板。
   - 内容模块间可「上一个 / 下一个」切换。
   - 抽屉打开状态用查询参数体现（如 `/article/:id?m=:moduleId`），可分享、浏览器后退键可关抽屉。

## 5. 内容模型（方案 C 的核心）

每篇文章 = 原文 + 内容模块数组；每个内容模块 = 若干「解读视角」，视角可挂自定义渲染。

```ts
type Category = 'code' | 'ai'
type LensType = 'intro' | 'concept' | 'reference' | 'pitfall'
// intro=导读 · concept=核心概念图解 · reference=指令交互速查 · pitfall=避坑

interface Article {
  id: string
  title: string
  category: Category
  cover?: string
  summary: string
  body: ContentBlock[]          // 原文，铺在总览页大纲下方
  contentModules: ContentModule[]
}

interface ContentModule {       // 第一层：来自文章最高层级标题
  id: string
  title: string
  order: number
  lenses: Lens[]                // 第二层：按需取用的解读视角
}

interface Lens {
  type: LensType
  title: string
  icon: string                  // Tabler outline 图标名
  content: unknown              // 结构化内容（按 type 而定）
  custom?: ReactNode            // 该视角专属交互组件（如 git 指令信息图）
}
```

- `custom` 是「每篇能玩出花」的口子：视角可塞专属交互组件（如可点击的 git 指令信息图挂在 reference 视角）。
- 该 schema 同时是**二期 AI 自动生成的目标格式**（AI 只需产出 JSON + 视角代码）。

### 5.1 解读视角菜单（4 选用，灵活取用）

每个内容模块的抽屉内，按需从以下视角中挑选并**连续排布**（顶部锚点导航 + scrollspy）：

1. **导读** intro：这段讲什么 · 为何该懂（类比可写进此处降门槛）
2. **核心概念图解** concept：把抽象概念图形化/类比讲清
3. **指令 / API 交互速查** reference：可点击弹气泡（重点交互）
4. **避坑** pitfall：常见错误与解决

> 演进记录：原 7 项菜单已收敛为内容模块下的 4 个解读视角。去掉「一句话总结、小测验」；「设计师类比、跟着做一遍」不再单设视角——类比融入导读/概念图解，实操步骤可融入相应视角正文。降门槛价值保留。

## 6. 交互范式库（可复用 + 可扩展）

交互范式是可复用的内容组件，**可嵌入任一解读视角的正文**（不是独立 tab）：

| 范式 | 常用视角 | 示例 |
|---|---|---|
| 可点击气泡 | 指令交互速查 | 点 `git push` 弹讲解+类比+易错点 |
| 设计师类比卡 | 导读 / 核心概念图解 | 「分支 = 设计稿副本」图文卡 |
| 分步实操清单 | 任一视角正文 | 可勾选的 checklist |
| 实时实验台 | AI 类 | Prompt 输入 → 看变化（接 API 为二期） |
| 拖拽 / 参数演示 | CSS 类 | 拖 Flex 参数看布局变化 |

- 允许每篇有专属范式，不强求统一。
- 小测验范式已移除。

## 7. 技术架构

- **栈**：React + Vite + TypeScript。
- **三层结构**：
  - `shell`：目录、路由、总览页（大纲+原文）、右侧抽屉、暗色主题。
  - `modules`：可复用的解读视角组件（导读/概念图解/交互速查/避坑）+ 交互范式组件 + 抽屉锚点导航（scrollspy）。
  - `content`：每篇文章数据（原文 + 内容模块 + 视角），含可选自定义组件。
- **路由**：
  - `/` 目录
  - `/article/:id` 信息图总览（基页）
  - `/article/:id?m=:moduleId` 模块详情以右侧抽屉覆盖在总览之上（非翻页）
- **内容来源**：
  - 一期：Claude 手动产出文章数据。
  - 二期入口（一期 UI 可放占位，点击提示"敬请期待"）：Cooper 链接 / Word·PDF 上传 / 粘贴网址。

### 7.1 建议目录结构

```
coding-learning/
  docs/                      # PRD、设计文档
  src/
    shell/                   # 布局、路由、主题、导航
    modules/                 # 可复用模块 + 交互范式组件
    content/                 # 每篇文章数据（+ 自定义组件）
    theme/                   # 设计 token
    App.tsx / main.tsx
  index.html
  vite.config.ts
  package.json
```

### 7.2 数据存储与文章扩充（一期不接数据库）

一期是**策展制**内容（由用户/Claude 精心产出，非用户实时上传），采用「文件即内容」的 **Git-based CMS**：

- 每篇文章 = 仓库里一个文件夹 `src/content/articles/<id>/`：
  - `index.ts`：文章元数据 + 原文 + 内容模块/视角结构化数据（即 Article schema）
  - 专属交互组件（如 git 信息图）放同目录 `.tsx`，挂到对应视角
  - 配图/封面放同目录 `assets/`
- `src/content/registry.ts`：汇总所有文章，供目录页使用。
- 全部进 git：天然有历史、可回滚、可 review。

**扩充一篇 = 加一个文件夹 + 注册一行**，构建时打包进站点。该方式可支撑数十到上百篇策展文章，零运维。

**何时才需要数据库（= 二期触发条件）：**

- 用户运行时自助上传/生成内容（构建后产生的内容无法进仓库）
- 多用户登录、权限、草稿态
- 大规模全文搜索、阅读数据统计

届时再加后端 + 数据库（如 Postgres），或上无头 CMS（Sanity/Strapi 等），或继续 Git-based 配 CI。Article schema 已对齐二期格式，迁移平滑。

## 8. UI 设计系统（暗色 Linear 风）

- **底色**：近黑 `#0d0d12`；卡片 `#14131c`。
- **强调色**：紫 `#6d5dfc`。
- **分类色**：代码类=紫；AI 类=青绿 `#5fd0a8`；警示=橙 `#f0a04b`。
- **字号**：延续用户锁定的 UI 字阶 24 / 14 / 12 / 11 / 10px；标题 22 / 18 / 16。
- **质感**：0.5px 细边框、柔和圆角、克制留白；推荐/选中项用 2px 紫边高亮。
- 字重仅用常规/中粗两档。
