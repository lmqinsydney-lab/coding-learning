# Coding Learning Platform · PRD / 设计文档

- 文档日期：2026-06-15
- 状态：已与用户对齐核心决策，待评审
- 负责人：用户（设计师，具备设计+coding 背景）
- 一句话定位：把代码 / AI 文章拆解、再生成为「可点击信息图」的学习 Web 平台，专为设计师降低学习门槛。

---

## 1. 背景与目标

团队鼓励设计师学习代码与 AI 知识。用户具备设计+coding 背景，希望建立一个学习平台，把团队成员的文章内容做**结构化拆解与再生成**，让设计师更容易理解代码/AI 知识。

文章内容分两类（一期）：

- **代码类**：用户自有或网上整理的代码文章，需站在设计师视角组织阅读与学习。
- **AI 学习类**：团队成员发布的 AI 学习/探索文章，或制作的好玩插件。

后续可能扩展业务类、设计类等分类（**非一期范围**）。

---

## 2. 核心决策（已与用户对齐）

| 维度 | 决策 |
|---|---|
| 模式 | 混合制：一期由 Claude 手动拆解+生成交互页，成员只「看/学」；二期做自助上传+AI 自动生成 |
| 一期形态 | 纯前端可跑的 Web 原型（`npm run dev` 即开），不接后端、不登录、不接 API |
| 最终形态 | 一个会持续长大的 Web 端产品 |
| 生成引擎 | LLM 生成可交互 HTML/SVG（非文生图位图）；效果不足再考虑文生图兜底 |
| 一期生成方式 | 由 Claude 手动产出文章数据与交互页 |
| 内容分类 | 仅「代码类」「AI 学习类」 |
| 架构 | 方案 C：共享外壳 + 可插拔模块 |
| 技术栈 | React + Vite + TypeScript |
| UI 风格 | 暗色高级感（参考 Linear 风，紫色强调） |

## 3. 一期范围（Scope）

**做：**

- 目录页（暗色卡片网格 + 分类筛选）
- 文章信息图总览页（模块平铺成一张信息图，可点击进入）
- 模块详情页（图文 + 专属交互）
- 可复用模块组件库 + 核心交互范式库
- 至少 2 篇样板文章（1 篇代码类 Git 文章 + 1 篇 AI 类）

**不做（YAGNI / 非目标）：**

- 后端、数据库、登录与权限
- API 调用、文生图
- 自助上传（Word/PDF/Cooper 链接）的解析与自动生成
- 部署上线
- 评论/收藏等社交功能
- 业务类/设计类分类

---

## 4. 信息架构与页面

三个层级：

1. **目录页** `/`
   - 暗色卡片网格；筛选 `全部 / 代码类 / AI 学习类`
   - 每张卡片：标题、分类、模块数、交互亮点标签、摘要
2. **文章信息图总览页** `/article/:id`
   - 整篇文章拆成 N 个模块，平铺成「信息图」，一眼看懂全貌
   - 点任一模块进入学习
3. **模块详情页** `/article/:id/:moduleId`
   - 图文 + 该模块专属交互
   - 模块间「上一个 / 下一个」串联学习

## 5. 内容模型（方案 C 的核心）

每篇文章 = 一份结构化数据 + 每个模块可挂自定义渲染。

```ts
type Category = 'code' | 'ai'

interface Article {
  id: string
  title: string
  category: Category
  cover?: string
  summary: string
  modules: Module[]
}

interface Module {
  id: string
  type: ModuleType
  title: string
  icon: string          // Tabler outline 图标名
  content: unknown      // 结构化内容（按 type 而定）
  custom?: ReactNode    // 该模块专属交互组件（如 git 指令信息图）
}
```

- `custom` 是「每篇能玩出花」的口子：模块可塞专属交互组件。
- 该 schema 同时是**二期 AI 自动生成的目标格式**（AI 只需产出 JSON + 模块代码）。

### 5.1 推荐模块菜单（7 选用，灵活取用）

每篇文章按内容从菜单中挑选模块组装，不强制全上：

1. 导读：这篇讲什么 · 为何设计师该学
2. 核心概念图解（类比讲清，图文卡片）
3. 指令 / API 交互速查（可点击弹气泡）
4. 设计师视角的类比
5. 跟着做一遍（分步实操清单）
6. 避坑 / 易错点
7. 一句话总结

> 注：原「总结+小测验」已按用户要求**去掉小测验**，仅保留「一句话总结」。

## 6. 交互范式库（可复用 + 可扩展）

| 范式 | 用途 | 示例 |
|---|---|---|
| 可点击气泡 | 指令/API 速查 | 点 `git push` 弹讲解+设计师类比+易错点 |
| 分步实操 | 跟着做一遍 | 可勾选的 checklist |
| 设计师类比卡 | 降门槛 | 「分支 = 设计稿副本」图文卡 |
| 实时实验台 | AI 类 | Prompt 输入 → 看变化（接 API 为二期） |
| 拖拽 / 参数演示 | CSS 类 | 拖 Flex 参数看布局变化 |

- 允许每篇有专属范式，不强求统一。
- 小测验范式已按用户要求移除。

## 7. 技术架构

- **栈**：React + Vite + TypeScript。
- **三层结构**：
  - `shell`：目录、路由、总览→模块导航、暗色主题。
  - `modules`：可复用模块组件库（对应推荐菜单 + 交互范式）。
  - `content`：每篇文章数据，含可选自定义组件。
- **路由**：
  - `/` 目录
  - `/article/:id` 信息图总览
  - `/article/:id/:moduleId` 模块详情
- **内容来源**：
  - 一期：Claude 手动产出文章数据。
  - 二期入口（一期 UI 可放占位，点击提示"敬请期待"）：Cooper 链接 / Word·PDF 上传 / 粘贴网址。

### 7.1 建议目录结构

```
coding-learning/
  docs/                      # PRD、设计文档
  src/
    shell/                   # 布局、路由、主题、导航
    modules/                 # 可复用模块 + 交互范式组件
    content/                 # 每篇文章数据（+ 自定义组件）
    theme/                   # 设计 token
    App.tsx / main.tsx
  index.html
  vite.config.ts
  package.json
```

## 8. UI 设计系统（暗色 Linear 风）

- **底色**：近黑 `#0d0d12`；卡片 `#14131c`。
- **强调色**：紫 `#6d5dfc`。
- **分类色**：代码类=紫；AI 类=青绿 `#5fd0a8`；警示=橙 `#f0a04b`。
- **字号**：延续用户锁定的 UI 字阶 24 / 14 / 12 / 11 / 10px；标题 22 / 18 / 16。
- **质感**：0.5px 细边框、柔和圆角、克制留白；推荐/选中项用 2px 紫边高亮。
- 字重仅用常规/中粗两档。
